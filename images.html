Having Images on Your Site

Your website is nothing without having some beautiful pictures or images on it. Now, you can not just slap some randon image on your site. Your image needs to be related to your content. Your image need to be clear or sharp enough to see. Your image needs to be big enough to see, but do not oversize it as well. To create images for web, it is best if you understand how the images work within the browser. Often, you need to find balance between the file size and quality. You can only use graphic file formats that browsers support.

Types of Images

JPEG: Joint Photographic Experts Group

  • Among the most widely used graphic formats in existence
  • Uses lossy compression to reduce file size
  • High levels of compression result in artifacts appearing in the image
  • Working with JPEGs as source images is not recommended

GIF: Graphics Interchange Format

  • Quickly became the most widely used image format on web
  • Support transparency and limited animation
  • GIFs are 8 bit graphics and are limited to a max of 256 colors
  • Are most widely used for icons and images with solid regions of color

PNG: Portable Network Graphics

  • Support transparency and have a lossless compression
  • Support millions of colors
  • Typically larger in file size than JPEGs and GIFs

SVG: Scalable Vector Graphics

  • Not widely used
  • Is a vector graphics format
  • Can be scaled independently of resolution
  • Graphics are written in SVG markup, which make them editable through code
